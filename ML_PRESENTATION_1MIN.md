# Выступление: ML-компоненты AI Finance Assistant
## Длительность: 1 минута (примерно 130-140 слов)

## Вариант 1: Технический акцент

Добрый день! Расскажу про ML-компоненты нашего приложения.

**Архитектура:** Используем российский GigaChat как основу AI-системы. Это Large Language Model, которая понимает естественный язык и генерирует персонализированные ответы.

**Ключевые ML-компоненты:**

Первое — **распознавание речи**. Интегрированы два API: GigaChat Audio и Sber Smart Speech. Пользователь говорит "Потратил 500 рублей на обед" — система распознает речь и передает текст дальше.

Второе — **извлечение транзакций**. Комбинация rule-based extraction и LLM. Сначала regex-паттерны извлекают суммы, категории, даты. Затем GigaChat структурирует данные и определяет тип транзакции с confidence score.

Третье — **анализ эмоций**. Fine-tuned промпты для GigaChat определяют эмоциональное состояние пользователя по 6 параметрам: радость, страх, гнев, грусть, удивление, нейтральность. Это позволяет адаптировать рекомендации под эмоциональный контекст.

Четвертое — **прогнозирование**. Статистический анализ паттернов расходов плюс LLM для интерпретации. Система моделирует сценарии: "Что будет, если уменьшить траты на 25%?" и показывает прогноз с расчетом экономии.

**Надежность:** Все компоненты имеют fallback-механизмы. При недоступности API система переходит на rule-based методы, обеспечивая непрерывную работу.

Спасибо!

## Вариант 2: Акцент на практической ценности ML

Добрый день! Расскажу, как машинное обучение делает наше приложение умным.

**Основа:** Российская LLM GigaChat — это "мозг" системы, который понимает естественный язык и генерирует персонализированные советы.

**ML-компоненты:**

**Распознавание речи** — пользователь говорит голосом, система преобразует речь в текст через Sber Smart Speech и GigaChat Audio API.

**Извлечение данных** — комбинация правил и AI. Пользователь пишет: "Потратил 500 рублей на обед" — система автоматически извлекает сумму, категорию, дату. Не нужно заполнять формы.

**Анализ эмоций** — уникальная фича. Система определяет эмоциональное состояние пользователя и адаптирует тон рекомендаций. Если пользователь тревожится — система успокаивает, если радуется — мягко напоминает о бюджете.

**Прогнозирование** — ML анализирует паттерны расходов и моделирует сценарии. Пользователь спрашивает: "Что будет, если уменьшу траты на 25%?" — получает прогноз с конкретными цифрами.

**Надежность:** Fallback-механизмы гарантируют работу даже при сбоях API.

Итог: ML превращает простое приложение в интеллектуального финансового консультанта.

## Вариант 3: Баланс техники и бизнес-ценности

Добрый день! Презентую ML-архитектуру нашего приложения.

**Технологический стек:** Российский GigaChat как ядро AI-системы. Это Large Language Model для понимания естественного языка и генерации ответов.

**Четыре ключевых ML-компонента:**

**1. Speech Recognition** — двойная интеграция: GigaChat Audio API и Sber Smart Speech. Поддержка 8 аудио-форматов. Пользователь говорит — система понимает.

**2. Transaction Extraction** — гибридный подход: rule-based extraction через regex плюс LLM для структурирования. Извлекаем тип, сумму, категорию, дату с confidence score. Адаптивный анализ: простые сообщения — краткий ответ, сложные — развернутый.

**3. Emotion Analysis** — промпт-инжиниринг для GigaChat. Анализ 6 эмоций с нормализацией распределения. Учет эмоций при финансовых рекомендациях — это наша уникальность.

**4. Forecast Generation** — статистика плюс LLM. Анализ паттернов расходов, моделирование сценариев, генерация прогнозов с расчетом экономии.

**Архитектура надежности:** Все компоненты имеют fallback на rule-based методы. Graceful degradation обеспечивает непрерывную работу.

**Результат:** ML превращает рутинный учет в интеллектуальный диалог с финансовым консультантом.

Спасибо!

## Типичные вопросы и ответы:

**Q: Почему GigaChat, а не GPT?**
A: Используем российские технологии, GigaChat отлично работает с русским языком, соответствует требованиям локализации данных.

**Q: Как обеспечивается надежность?**
A: Все ML-компоненты имеют fallback на rule-based методы. При недоступности API система продолжает работать с базовой функциональностью.

**Q: Какая точность извлечения транзакций?**
A: Используем confidence score для каждой транзакции. Комбинация rule-based и LLM дает высокую точность, особенно для простых случаев.

**Q: Как обрабатываются ошибки ML?**
A: Многоуровневая система: retry логика, fallback на простые методы, graceful degradation. Пользователь всегда получает ответ.

## Альтернативный вариант (если нужно больше технических деталей):

Добрый день! ML-архитектура нашего приложения построена на российской LLM GigaChat.

**Архитектура:** Трехуровневая система с автоматическим выбором модели — GigaChat, GigaChat-Plus, GigaChat-Pro. Кэширование токенов и оптимизация использования.

**Компоненты:**

**Speech Recognition:** Двойная интеграция — GigaChat Audio API (primary) и Sber Smart Speech (fallback). Поддержка PCM16, WAV, MP3, OGG Opus. Таймауты 15-30 секунд.

**Emotion Analysis:** Fine-tuned промпты с извлечением параметров. 6 эмоций с нормализацией распределения. Учет финансовых целей, опыта, ситуации при анализе.

**Forecast Generation:** Статистический анализ паттернов расходов по категориям. LLM извлекает параметры прогноза из запроса. Генерация месячных прогнозов с расчетом экономии.

**Надежность:** Fallback на rule-based методы для всех компонентов. Graceful degradation. Обработка таймаутов и ошибок API.

**Результат:** ML превращает приложение в интеллектуального финансового консультанта с пониманием естественного языка и эмоций.
